# 简化版游戏玩法重设计 Spec

## Why

目前的游戏玩法过于复杂，需要玩家在多个页面之间频繁切换操作。用户希望的是一个更轻松的放置类/模拟经营体验：

* 核心操作仅需"点击下一天"

* 其他操作尽量减少

* 让玩家专注于俱乐部发展的宏观决策

## 现状分析

### 现有系统梳理

#### 1. 阵容系统 (team/index.vue)

**当前玩法**:

* 手动查看一线队/青训队

* 手动训练每个选手的每个属性

* 手动提拔青训选手

* 手动解约选手

**问题**: 训练操作过于繁琐，需要逐个选手逐个属性点击

#### 2. 转会系统 (transfer/index.vue)

**当前玩法**:

* 主动浏览转会市场

* 位置筛选

* 手动刷新市场

* 手动签约

* 青训挖掘

**问题**: 需要主动关注转会市场，容易错过好选手

#### 3. 比赛系统 (match/index.vue)

**当前玩法**:

* 手动开始每场比赛

* 可选择跳过

* 查看战报需要点击进入

**问题**: 每场比赛都要手动操作，联赛比赛过多时很繁琐

#### 4. 联赛系统 (league/index.vue)

**当前玩法**:

* 查看积分榜

* 查看赛程

* 手动模拟本轮比赛

**问题**: 需要手动触发比赛模拟

#### 5. 赞助商系统 (sponsor/index.vue)

**当前玩法**:

* 查看可用赞助商

* 手动签约

* 查看合同状态

**问题**: 赞助商到期容易忘记续约

#### 6. 粉丝系统 (fans/index.vue)

**当前玩法**:

* 查看粉丝数据

* 查看声望等级

* 查看历史趋势

**问题**: 纯展示，缺乏互动

***

## 新玩法设计

### 核心原则

1. **一键推进**: 点击"下一天"按钮推进游戏
2. **自动处理**: 日常事务自动完成
3. **智能提醒**: 重要事项主动提醒
4. **集中展示**: 所有关键信息在首页展示

### UI布局重构

```
┌─────────────────────────────────────┐
│  [俱乐部名]    [资金] [粉丝] [声望]   │  ← 顶部状态栏（保持现有）
├─────────────────────────────────────┤
│                                     │
│  ┌─────────┐  ┌─────────┐          │
│  │ 阵容卡片 │  │ 赛程卡片 │          │  ← 左右滑动切换卡片
│  │ (左滑)  │  │ (右滑)  │          │
│  └─────────┘  └─────────┘          │
│                                     │
│        ┌─────────┐                 │
│        │  比赛球  │                 │  ← 下一天按钮（球体设计）
│        │  (点击) │                 │
│        └─────────┘                 │
│                                     │
│  ┌─────────────────────────────┐   │
│  │      待处理事项列表          │   │  ← 红点提醒区域
│  │  • 赞助商即将到期           │   │
│  │  • 发现潜力选手             │   │
│  │  • 选手合同到期提醒         │   │
│  └─────────────────────────────┘   │
│                                     │
├─────────────────────────────────────┤
│  [首页] [阵容] [比赛] [更多]        │  ← 底部导航（简化为4个）
└─────────────────────────────────────┘
```

### 卡片设计（左右滑动切换）

只保留2个主卡片：

1. **阵容卡片** - 包含阵容/转会/青训功能（Tab切换）
2. **赛程卡片** - 包含赛程/战绩/联赛功能（Tab切换）

其他功能（赞助商、粉丝、设施）集成到"更多"页面

## 自动化程度分级

### 1. 完全自动化（无需玩家操作）

| 系统   | 自动行为       | 日报展示      |
| ---- | ---------- | --------- |
| 训练   | 每周自动训练所有选手 | 训练成果摘要    |
| 比赛   | 联赛比赛自动进行   | 比赛结果+战报链接 |
| 体力恢复 | 每天自动恢复     | 无（后台处理）   |
| 周收支  | 每周一自动结算    | 收支明细      |
| 粉丝增长 | 根据战绩自动增长   | 粉丝变化      |

### 2. 配置后自动化（一次配置，自动执行）

| 系统   | 配置项       | 自动行为         | 提醒时机       |
| ---- | --------- | ------------ | ---------- |
| 训练策略 | 保守/平衡/激进  | 根据策略自动安排训练强度 | 策略可在设置修改   |
| 战术风格 | 进攻/平衡/防守  | 自动BP和战术选择    | 重要比赛前可手动调整 |
| 转会偏好 | 年龄/潜力/即战力 | 主动推荐符合偏好的选手  | 发现好选手时提醒   |

### 3. 智能提醒后手动操作（系统提醒，玩家决策）

| 场景             | 提醒方式    | 操作选项         |
| -------------- | ------- | ------------ |
| 赞助商合同到期（30天内）  | 首页红点+弹窗 | 续约/寻找新赞助商/忽略 |
| 发现高潜力选手        | 首页卡片展示  | 签约/忽略        |
| 主力选手合同到期（30天内） | 首页红点    | 续约/放走        |
| 选手受伤需要替补       | 弹窗提醒    | 调整阵容/使用替补    |
| 重要比赛（季后赛/决赛）   | 比赛前提醒   | 手动调整/自动进行    |
| 资金不足发薪         | 紧急弹窗    | 紧急融资/出售选手    |

### 4. 必须手动操作（无法自动化）

| 操作     | 触发场景      | 入口位置       |
| ------ | --------- | ---------- |
| 签约新选手  | 转会期+发现目标  | 阵容卡片-转会Tab |
| 解约选手   | 需要腾出名额    | 阵容卡片-选手详情  |
| 设施升级   | 资金充足时     | 更多页面-设施    |
| 赞助商签约  | 合同到期或主动更换 | 更多页面-赞助商   |
| 应对随机事件 | 事件触发时     | 弹窗选择       |

## 详细功能设计

### 1. 下一天按钮（核心交互）

**设计**:

* 球体设计，位于底部卡片上方居中

* 点击后有按压动画

* 长按可连续推进多天

**行为**:

* 推进1天

* 检查并触发当日事件

* 如果是周一，触发周结算

* 显示日报弹窗

### 2. 日报系统

**触发时机**: 每次点击"下一天"

**日报内容**:

```
📅 第X天 - 2024年X月X日

🎮 比赛结果
   vs XXX队  2:1 胜
   [查看战报]

💰 资金变动
   收入: +50万（比赛奖励）
   支出: -30万（选手薪资）
   净变化: +20万

👥 粉丝变化
   +150人

📊 选手状态
   张三 体力恢复至85%
   李四 状态良好

⚠️ 待处理事项
   • 王五合同即将到期（28天）
```

### 3. 周结算系统

**触发时机**: 每周一（点击下一天时自动触发）

**周报告内容**:

```
📊 第X周总结

🏆 本周战绩
   2胜1负  胜率66%
   联赛排名: 第3名

💰 本周收支
   收入: 200万
     - 赞助商: 100万
     - 粉丝收入: 50万
     - 比赛奖励: 50万
   支出: 150万
     - 选手薪资: 120万
     - 设施维护: 30万
   净收益: +50万

📈 选手成长
   张三 操作+2 意识+1
   李四 团队+3

🎯 训练成果
   本周训练强度: 平衡
   总体成长: 良好

⚠️ 重要提醒
   • 赞助商合同还有15天到期
   • 发现潜力选手：赵六（潜力92）
```

### 4. 阵容卡片（左卡片）

**Tab设计**:

* Tab1: 阵容 - 显示主力5人+替补，快速查看状态

* Tab2: 转会 - 显示系统推荐的选手+主动搜索

* Tab3: 青训 - 显示青训选手+挖掘按钮

**阵容Tab**:

* 5个主力位置展示（头像+名字+实力+体力条）

* 点击选手打开详情（可手动训练/休息/解约）

* 体力低于30%显示警告

**转会Tab**:

* "系统推荐"区域（智能推荐的3个选手）

* "主动搜索"按钮（进入详细转会市场）

* 转会期倒计时显示

**青训Tab**:

* 当前青训选手列表

* "挖掘新选手"按钮

* 青训潜力预览

### 5. 赛程卡片（右卡片）

**Tab设计**:

* Tab1: 赛程 - 显示未来3场比赛

* Tab2: 战绩 - 显示最近5场比赛结果

* Tab3: 联赛 - 积分榜+排名

**赛程Tab**:

* 下一场比赛高亮显示

* 显示对手实力和近期状态

* 如果是今天比赛，显示"即将开始"

**战绩Tab**:

* 最近5场比赛结果

* 胜负走势图表

* 场均数据

**联赛Tab**:

* 当前排名

* 前5名积分榜

* 距离上一名/下一名的差距

### 6. 待处理事项系统

**展示位置**: 首页底部列表

**事项类型**:

1. 紧急（红色）: 资金不足、主力受伤无替补
2. 重要（黄色）: 合同30天内到期、赞助商到期
3. 提示（蓝色）: 发现潜力选手、可升级设施

**交互**:

* 点击事项跳转到对应处理页面

* 可滑动删除已读事项

* 红点数字显示未处理数量

## 技术实现要点

### 1. 卡片滑动实现

* 使用swiper或类似库实现左右滑动

* 底部指示器显示当前卡片位置

* 支持手势滑动和点击切换

### 2. 日报/周报弹窗

* 使用modal组件

* 支持点击外部关闭

* 重要信息高亮显示

### 3. 自动系统触发

* 在gameStore的advanceTime中统一处理

* 使用flag标记避免重复触发

* 异步处理比赛模拟避免卡顿

### 4. 智能推荐算法

* 转会推荐: 根据俱乐部需求（缺什么位置）+ 资金情况

* 训练策略: 根据选手年龄和潜力自动调整

* 战术选择: 根据对手实力自动调整
