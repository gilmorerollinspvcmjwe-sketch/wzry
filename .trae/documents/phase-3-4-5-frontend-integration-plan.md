# Phase 3/4/5 前端集成详细设计计划（简化版）

## 核心设计原则

1. **英雄系统简化**：只展示核心信息，服务于选手系统和比赛计算
2. **选手系统为核心**：英雄是选手的"工具"，重点在选手偏好和使用统计
3. **比赛计算独立**：BP、胜负计算是独立系统，与展示层分离

---

## 一、英雄系统（简化版）

### 1.1 英雄基础信息
每个英雄只保留核心字段：
```typescript
interface Hero {
  id: string;
  name: string;           // 英雄名称
  role: HeroRole;         // 定位（坦克/战士/刺客/法师/射手/辅助）

  counters: string[];    // 克制英雄ID列表

}
```

## 二、选手系统（核心）

### 2.1 选手与英雄的关系

```typescript
interface Player {
  // 基础信息
  id: string;
  name: string;
  gameId: string;
  age: number;
  position: Position;        // 擅长位置（对抗路/打野/中路/发育路/游走）
  
  // 英雄偏好
  preferredRoles: HeroRole[];  // 偏好的英雄定位（如["坦克", "战士"]）
  preferredTags: string[];     // 偏好的标签（如["控制", "爆发"]）
  favoriteHeroes: string[];    // 常用英雄ID列表（top 5）
  
  // 英雄使用统计
  heroStats: Map<string, {    // heroId -> 统计数据
    games: number;            // 使用场次
    wins: number;             // 胜场
    winRate: number;          // 胜率
  }>;
  
  // 五维属性
  stats: {
    mechanics: number;    // 操作（影响英雄上限发挥）
    awareness: number;    // 意识（影响团战判断）
    teamwork: number;     // 团队（影响配合）
    mentality: number;    // 心态（影响逆风表现）
    stamina: number;      // 体力（影响比赛状态）
  };
  
  // 性格特质（影响比赛计算）
  personality: {
    playStyle: 'aggressive' | 'stable' | 'flexible';  // 比赛风格
    pressureResistance: 'clutch' | 'fragile' | 'normal'; // 抗压能力
  };
  
  // 特殊天赋（影响比赛计算）
  talents: string[];  // 如["关键先生", "英雄海"]
}
```

## 三、比赛计算系统（独立）

### 3.1 比赛计算要素

比赛胜负计算由以下因素决定：

```typescript
interface MatchCalculation {
  // 1. 选手实力
  playerPower: number;      // 五维属性综合值
  
  // 2. 英雄匹配度
  heroFit: number;          // 选手偏好 vs 使用英雄匹配度
  // - 偏好定位匹配: +10%
  // - 偏好标签匹配: +5%
  // - 常用英雄: +5%
  
  // 3. BP因素
  bpAdvantage: number;      // BP优劣（克制关系）
  // - 英雄被克制: -15%
  // - 英雄克制对方: +15%
  
  // 4. 性格影响
  personalityBonus: number;
  // - 激进风格 + 顺风局: +10%
  // - 大心脏 + 决胜局: +15%
  // - 玻璃心 + 逆风局: -10%
  
  // 5. 天赋影响
  talentBonus: number;
  // - 关键先生 + 决胜局: +20%
  // - 英雄海: 英雄匹配度额外+5%
  
  // 6. 体力状态
  staminaFactor: number;    // 0.8 - 1.2
  
  // 7. 随机因素
  randomFactor: number;     // ±10% 随机波动
}
```

## 六、数据联动

### 6.1 核心数据流

```
1. 选手训练 → 属性提升
   ↓
2. 选择英雄 → 匹配度计算
   ↓
3. BP选择 → 克制关系计算
   ↓
4. 比赛计算 → 胜负结果
   ↓
5. 战报生成 → 更新统计数据
   ↓
6. 段位/胜率更新 → 选手价值变化
```
